## Integrated-stack checks

The review followed actual repeater code paths rather than evaluating core in isolation. Relevant integrated behavior includes:

- RepeaterHandler flood and direct forwarding;
- TraceHelper TRACE validation and forwarding;
- ACL login timestamp replay protection;
- PacketRouter payload dispatch and forwarding order;
- companion bridge and frame-server command handling;
- SQLite message, contact, channel, and preference persistence;
- duty-cycle, LBT, TX serialization, and advert scheduling.

## Publication verification pass

- All retained reports were checked for direct source support in the supplied OpenHop and MeshCore snapshots.
- All **217 embedded source excerpts** were compared with their referenced local files. Every numbered source line matched; one outer Markdown fence remains intentionally widened around BUG-027 because the excerpt itself contains an internal fence.
- BUG-004, BUG-005, and BUG-013 were narrowed to match the exact active control flow and avoid broader claims than the evidence supports.

## Test execution note

The OpenHop Core test run reached 794 passing tests before encountering failures in supplied-snapshot hardware CAD validation. Those failures are outside the retained compatibility claims, but they mean this audit does not claim the complete upstream test suite is green. The publication decision is based on source-level comparison and targeted control-flow validation, not a blanket test-suite pass.

## Runtime validation still recommended

For release-level compatibility confidence, add:

- byte-for-byte packet fixtures generated by both MeshCore and OpenHop;
- companion protocol tests against strict request/response correlation;
- one-, two-, and three-byte path-hash multi-node routing tests;
- hash-collision tests across remote contacts and local virtual identities;
- restart tests for every queued-message variant;
- RF or hardware-in-the-loop tests for airtime, duty cycle, LBT, and retransmit scheduling;
- configuration-migration tests proving optional advert limiting remains disabled when fields are absent.

## Interpretation

“Source-confirmed” means the incompatible or incorrect control flow is directly present in the reviewed source. It does not claim that every consequence has been reproduced on physical radios.
