## What happens

SET_OTHER_PARAMS is backward-compatible: a short frame updates manual-add only, and omitted newer fields remain unchanged. OpenHop supplies zero for every omitted field, resetting telemetry modes, location policy, and multi-ACKs.

## How official MeshCore handles it

The firmware always sets manual_add_contacts, conditionally sets telemetry modes only when present, then conditionally sets advert_loc_policy and multi_acks at their respective lengths.

## How the OpenHop stack handles it

**OpenHop Core:** _cmd_set_other_params assigns telemetry_modes, advert_loc_policy, and multi_acks to zero whenever their data bytes are absent, then passes all four to set_other_params.

## What needs to change

Read current preferences and update each optional field only when its byte is present. Add tests for data lengths one through four.
