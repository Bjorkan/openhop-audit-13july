## What happens

The repeater selects one occurrence threshold for each minimal, moderate, or strict loop-detection mode regardless of whether path hashes are one, two, or three bytes wide.

## How official MeshCore handles it

MeshCore uses width-aware thresholds: minimal is 4/2/1, moderate is 2/1/1, and strict is 1/1/1 for one-/two-/three-byte hashes.

## How the OpenHop stack handles it

**OpenHop Repeater:** Minimal mode permits four occurrences even for two- and three-byte hashes. Moderate mode permits two where MeshCore stops after one. Loops can therefore continue longer than intended.

## What needs to change

Replace `LOOP_DETECT_MAX_COUNTERS` with a mode-to-hash-width-to-threshold table and explicitly reject the reserved four-byte width. Test every mode and width at threshold minus one and at threshold.
