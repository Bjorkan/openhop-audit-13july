# BUG-099 — Path-discovery pushes replace encoded path lengths with raw byte counts

[← Active finding list](../README.md#active-findings)

| Field | Value |
|---|---|
| Severity | **High** |
| Area | Companion path discovery |
| Affected components | OpenHop Core |
| Status | Confirmed from the supplied source snapshots |

## TL;DR

MeshCore path lengths encode both hop count and one-, two- or three-byte hash width. OpenHop extracts the path bytes but discards both encoded length bytes, then builds the app push using `len(path)`. A two-hop path using two-byte hashes is therefore announced as four one-byte hops, corrupting route interpretation in the companion app.

**Current status: 🔴 Not fixed.** Rechecked against latest supplied Core head [`abe1d85`](https://github.com/openhop-dev/openhop_core/commit/abe1d857734d5d1f549ada27af6f95c7e5e53e95) and Repeater head [`b62960f`](https://github.com/openhop-dev/openhop_repeater/commit/b62960fa3041447bedadb2131d892a6565d7e519), as applicable. The three new Core commits and two new Repeater commits do not remove this mismatch; the complete regression suites and focused changed-path review found no contrary behavior.

## What happens

MeshCore path lengths encode both hop count and one-, two- or three-byte hash width. OpenHop extracts the path bytes but discards both encoded length bytes, then builds the app push using `len(path)`. A two-hop path using two-byte hashes is therefore announced as four one-byte hops, corrupting route interpretation in the companion app.

## How official MeshCore handles it

The firmware validates and forwards the original encoded `out_path_len` and `in_path_len` bytes, while `Packet::writePath()` copies the corresponding number of route bytes.

## How the OpenHop stack handles it

The protocol-response handler calculates the output path's byte count and stores only raw byte strings in `path_info`; the inbound packet's encoded path length is also omitted. The push callback reconstructs both fields from Python byte-string lengths, which cannot recover hash width.

## What needs to change

Carry `(encoded_len, path_bytes)` for both directions through the response callback and serialize the original encoded length bytes. Add tests for all three hash widths, zero-hop paths and invalid encoded lengths.
