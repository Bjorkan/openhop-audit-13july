## TL;DR

Preserve explicit operator opt-in and document that enabled anti-abuse policy intentionally diverges from MeshCore forwarding.

## What happens

Advert limiting is an intentional optional anti-abuse policy, and the supplied example configuration disables the token bucket, penalty box, and adaptive limiter. The implementation defaults each missing `enabled` field to `True`, so installations with older or partial configuration files silently activate a forwarding-blocking policy.

## How official MeshCore handles it

A simple MeshCore repeater forwards each valid unseen advert that passes general region, hop, and loop rules. It has no implicit per-public-key token bucket.

## How the OpenHop stack handles it

**OpenHop Repeater:** When limiter sections are absent, `AdvertHelper` enables rate limiting, penalties, and adaptive behavior. A policy rejection marks the packet do-not-retransmit, affecting RF forwarding rather than only local storage or metrics. This contradicts the compatibility-oriented example defaults.

## What needs to change

Default every optional advert-limiter feature to disabled when configuration is absent. Preserve explicit operator opt-in and document that enabled anti-abuse policy intentionally diverges from MeshCore forwarding. Prefer separating local storage throttling from RF forwarding where practical.
