# BUG-059 — Local one-byte identity collisions overwrite routing and persistence state

[← Bug list](../README.md#bug-list)

| Field | Value |
|---|---|
| Severity | **High** |
| Area | Repeater multi-identity lifecycle |
| Affected components | OpenHop Repeater |
| Status | Confirmed from the supplied source snapshots |

## TL;DR

Room, companion, and repeater identities are indexed in several dictionaries by `public_key[0]`. Startup does not validate `companion_hash` before assignment, starts the server before checking the IdentityManager result, and ignores a `False` return from `register_identity`. To fix it, prevalidate all local identities, including the default repeater, before creating any helper, database namespace, or server. Fail startup clearly on collision, or redesign indexes as lists or full-key namespaces with authenticated fan-out.

## What happens

Room, companion, and repeater identities are indexed in several dictionaries by `public_key[0]`. Startup does not validate `companion_hash` before assignment, starts the server before checking the IdentityManager result, and ignores a `False` return from `register_identity`. The default repeater identity is not included in the same collision check.

## How official MeshCore handles it

Official MeshCore stores one local identity in `Mesh::self_id`, so two local identities cannot compete for the same short destination prefix inside one firmware instance. On receive, the destination prefix only selects the local node and matching peer candidates; the packet is treated as locally authenticated only after `MACThenDecrypt` succeeds.

## How the OpenHop stack handles it

**OpenHop Repeater:** A configured collision can overwrite `companion_bridges[hash]`, share the same `0xHH` SQLite namespace, replace text or login handlers, and leave another already-started TCP server inconsistent.

## What needs to change

Prevalidate all local identities, including the default repeater, before creating any helper, database namespace, or server. Fail startup clearly on collision, or redesign indexes as lists or full-key namespaces with authenticated fan-out. Apply the same rule to startup and hot reload.
