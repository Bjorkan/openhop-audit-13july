## What happens

Packet.read_from() accepts a frame containing only the header, encoded path length, and path. Official MeshCore rejects the same wire bytes because every packet must contain at least one payload byte. The invalid packet can therefore reach raw subscribers and generic Core integrations as a successfully parsed Packet before a downstream handler happens to reject it.

## How official MeshCore handles it

After reading the path, Packet::readFrom() explicitly checks i >= len and returns false. This makes an empty payload a malformed wire frame.

## How the OpenHop stack handles it

After reading the path, Packet.read_from() assigns data[idx:] to payload and validates only the maximum size. Zero is accepted. A focused reproduction accepted a two-byte flood ACK frame and reported payload_len == 0.

## What needs to change

Require at least one remaining byte after the path before assigning payload. Add zero-payload vectors for every route form, including transport routes, and verify the dispatcher does not invoke parsed-packet callbacks for them.
