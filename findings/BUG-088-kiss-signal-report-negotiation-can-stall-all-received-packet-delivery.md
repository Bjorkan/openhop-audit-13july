# BUG-088 — KISS signal-report negotiation can stall all received packet delivery

## TL;DR

The official modem answers SET_SIGNAL_REPORT with the GET_SIGNAL_REPORT response code (0x9A), but OpenHop waits for SET|0x80 (0x99) or generic OK. The valid response is queued as unrelated and the setter times out. Independently, OpenHop always withholds DATA frames until RX_META arrives; when reporting is disabled, official firmware intentionally sends no RX_META, so packets accumulate and are eventually dropped.

**Current status: 🔴 Not fixed.** Confirmed in the supplied Core snapshot [`9ea7269`](https://github.com/openhop-dev/openhop_core/commit/9ea7269a7e7e903fe433b1f952a4026fe3dcc81b) and Repeater snapshot [`6aafa7f`](https://github.com/openhop-dev/openhop_repeater/commit/6aafa7fe991b5b3199b18149f84417f8522d94b2), as applicable. ## What happens

The official modem answers SET_SIGNAL_REPORT with the GET_SIGNAL_REPORT response code (0x9A), but OpenHop waits for SET|0x80 (0x99) or generic OK. The valid response is queued as unrelated and the setter times out. Independently, OpenHop always withholds DATA frames until RX_META arrives; when reporting is disabled, official firmware intentionally sends no RX_META, so packets accumulate and are eventually dropped.

## How official MeshCore handles it

KissModem sends every DATA frame immediately and appends RX_META only while signal reporting is enabled. SET_SIGNAL_REPORT responds with `HW_RESP(HW_CMD_GET_SIGNAL_REPORT)` and the current enabled byte.

## How the OpenHop stack handles it

The method-level response check allows 0x9A, but `_send_command()` never considers it acceptable for subcommand 0x19. The receive parser has no disabled-report mode and only dispatches queued DATA from the RX_META branch.

## What needs to change

Map SET_SIGNAL_REPORT to the 0x9A response, retain the negotiated enabled state, and dispatch DATA immediately with unavailable/sentinel metrics when reports are disabled. On reconnect, query the mode before deciding whether DATA must wait for metadata.
