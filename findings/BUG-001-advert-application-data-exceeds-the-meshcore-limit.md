## TL;DR

OpenHop permits 96 bytes of application data in an advert, while MeshCore reserves at most 32 bytes. An advert accepted or produced by OpenHop can therefore be rejected, truncated, or parsed differently by MeshCore nodes. To fix it, change the OpenHop limit to 32 bytes and reject longer inbound and outbound application data. Test lengths 32 and 33 on both encode and decode paths.

**Current status: ✅ Fully fixed.** OpenHop now uses the 32-byte advert application-data limit and rejects oversized outbound adverts. The inbound handler deliberately truncates application data to 32 bytes before signature verification, which matches official MeshCore rather than leaving an unresolved decode-path mismatch; see [AdvertHandler](https://github.com/openhop-dev/openhop_core/blob/fix/all-the-things-core/src/openhop_core/node/handlers/advert.py#L77-L87). The limit correction was introduced in [`061a5da` — `fix(advert): cap advert application data at 32 bytes`](https://github.com/openhop-dev/openhop_core/commit/061a5da). Rechecked against current Core head [`9ea7269`](https://github.com/openhop-dev/openhop_core/commit/9ea7269a7e7e903fe433b1f952a4026fe3dcc81b), which is a version-only bump after [`9355d08`](https://github.com/openhop-dev/openhop_core/commit/9355d08e21423886a17979c0d8defb891f5d9d72), and current Repeater head [`9688fe7`](https://github.com/openhop-dev/openhop_repeater/commit/9688fe70e44d85f3cec7de5c77dd804d915a7506), whose only later change after [`6aafa7f`](https://github.com/openhop-dev/openhop_repeater/commit/6aafa7fe991b5b3199b18149f84417f8522d94b2) adds observation-only neighbour-link metrics; no regression was found.

## What happens

OpenHop permits 96 bytes of application data in an advert, while MeshCore reserves at most 32 bytes. An advert accepted or produced by OpenHop can therefore be rejected, truncated, or parsed differently by MeshCore nodes.

## How official MeshCore handles it

MeshCore defines MAX_ADVERT_APP_DATA_SIZE as 32 and builds/parses advertisements around that bound.

## How the OpenHop stack handles it

**OpenHop Core:** OpenHop defines MAX_ADVERT_APP_DATA_SIZE as 96 and uses that value in its advert model and builder validation.

## What needs to change

Change the OpenHop limit to 32 bytes and reject longer inbound and outbound application data. Test lengths 32 and 33 on both encode and decode paths.
