## What happens

After Core has already terminated the authenticated C-string at the first NUL, Repeater calls a second unrestricted `.rstrip()` before room handling. Valid trailing spaces, tabs and other Unicode whitespace are removed from posts and CLI payloads. Official MeshCore copies the C string as supplied and preserves trailing whitespace.

## How official MeshCore handles it

The room server passes `&data[5]` to `addPost()`, which uses `StrHelper::strncpy()` up to the fixed message limit. The C-string terminator bounds the text, but trailing non-NUL bytes are retained.

## How the OpenHop stack handles it

`message_text.rstrip('\x00').rstrip()` first removes trailing NULs and then removes all whitespace. Core’s `_text_len()` has already excluded padding, so the second operation changes authenticated content rather than cleaning protocol padding.

## What needs to change

Remove the generic whitespace trim. Preserve exactly the decoded bytes before the first NUL; perform command-specific token normalization only inside the CLI parser if required. Add round-trip tests for spaces, tabs and non-ASCII whitespace.
