## TL;DR

The dispatcher classifies an ADVERT as local when only the first public-key byte matches, and classifies almost every other non-group packet as local when payload byte 1 matches. Those bytes are not a universal sender field. Valid packets can therefore be discarded before authentication or payload-specific parsing.

## What happens

The dispatcher classifies an ADVERT as local when only the first public-key byte matches, and classifies almost every other non-group packet as local when payload byte 1 matches. Those bytes are not a universal sender field. Valid packets can therefore be discarded before authentication or payload-specific parsing.

## How official MeshCore handles it

MeshCore uses packet-type-aware ownership and destination checks. Advert identity is based on the advertised public key, and packets without a sender-prefix field are not subjected to a generic payload-byte ownership rule.

## How the OpenHop stack handles it

`_is_own_packet()` compares one byte for adverts and `payload[1]` for all remaining non-group types. ACK, multipart, trace, control and raw/custom layouts do not share a universal sender byte at that position.

## What needs to change

Remove the generic payload-byte heuristic. Mark locally transmitted packet hashes as seen, and use type-specific authenticated identity checks only where the wire format actually carries a sender identity.
